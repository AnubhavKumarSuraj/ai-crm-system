# Customer Detail Page (High-Impact View)

## 🎯 Goal
Create a premium customer detail page that provides full context, insights, and actionable controls.  
This page should feel like a **customer intelligence dashboard**, not just a data view.

---

## 🧩 Layout Structure

Two-column layout:

LEFT (Main Content)
- Customer Header
- Key Information
- Activity Timeline
- Message Preview

RIGHT (Sidebar)
- Action Buttons
- AI Insight Box

---

## 💎 1. Customer Header

Display:
- Customer Name + Avatar
- Phone Number
- Status Badge (Inactive / High Risk)
- "Inactive for X days"

Purpose:
- Quick identity + state awareness

---

## 💎 2. Key Information Section

Show important metrics:

- Last Visit Date
- Days Inactive
- Risk Level
- Last Contacted

Example:
Last visit: 22 Jan 2026  
Inactive: 101 days  
Risk: High  
Last contacted: Never  

Purpose:
- Immediate understanding of customer condition

---

## 💎 3. Activity Timeline

Vertical timeline showing events:

Examples:
- Jan 22 – Last visit  
- Feb 10 – Became inactive  
- Mar 01 – Recovery campaign sent  
- Mar 02 – No response  

Guidelines:
- Use icons for clarity
- Keep spacing clean
- Chronological order

Purpose:
- Storytelling of customer journey

---

## 💎 4. Message Preview

Show WhatsApp-style interaction:

- Sent message (from business)
- Customer reply (if any)

Purpose:
- Context of communication
- Visual engagement

---

## 💎 5. Right Panel (Actions)

Primary Action:
- ⚡ Recover Now

Secondary Actions:
- Edit Customer
- Delete Customer

Purpose:
- Fast decision making

---

## 💎 6. AI Insight Box

Display helpful suggestions:

Examples:
- "High chance to return with discount"
- "Best time to message: Evening (6–9 PM)"

Purpose:
- Make the product feel intelligent
- Guide user action

---

## 🎨 Design Guidelines

- Clean card-based layout
- Soft shadows
- Rounded corners
- Consistent spacing
- Minimal clutter
- Focus on readability

---

## 🧠 Final Outcome

This page should feel like:
- A decision engine
- A customer intelligence system
- A premium SaaS experience

NOT:
- A static data page
